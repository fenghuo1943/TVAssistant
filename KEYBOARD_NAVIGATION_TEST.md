# 网址管理页面键盘导航测试指南

## 测试场景

### 场景 1：从主页进入设置页面
1. 启动应用
2. 点击右上角齿轮图标进入设置
3. 默认聚焦在侧边栏第一个项（常规）

### 场景 2：导航到网址管理菜单
**操作：**
- 按 **↓** 键，聚焦到"网址管理"
- 按 **右键** 或 **Enter** 键

**预期结果：**
- 右侧内容区显示网址列表
- 第一个网址项（TV 直播）获得焦点
- 网址项显示高亮边框和背景

### 场景 3：在网址列表中导航
**操作：**
- 按 **↓** 键

**预期结果：**
- 焦点从"TV 直播"移动到"央视片库"
- "央视片库"项高亮显示

**操作：**
- 继续按 **↓** 键，遍历所有网址项
- 按 **↑** 键向上移动

**预期结果：**
- 焦点在所有 4 个网址项之间循环移动
- 到达最后一项后，再按 **↓** 会回到侧边栏
- 到达第一项后，再按 **↑** 会回到侧边栏

### 场景 4：进入按钮层级
**操作：**
1. 聚焦在任意网址项（如"抖音"）
2. 按 **右键**

**预期结果：**
- 该网址项的"添加"或"移除"按钮获得焦点
- 按钮显示高亮效果

### 场景 5：在按钮之间导航
**操作：**
- 在按钮聚焦状态下，按 **↓** 键

**预期结果：**
- 焦点移动到下一个网址项的按钮
- 例如：从"抖音"的按钮移动到"腾讯视频"的按钮

**操作：**
- 按 **↑** 键

**预期结果：**
- 焦点移动到上一个网址项的按钮
- 如果当前是第一个按钮，按 **↑** 会回到侧边栏

### 场景 6：返回网址项层级
**操作：**
- 在按钮聚焦状态下，按 **左键**

**预期结果：**
- 焦点返回到当前网址项
- 网址项重新获得焦点

### 场景 7：返回侧边栏
**操作：**
- 在网址项或按钮聚焦状态下，按 **左键**

**预期结果：**
- 焦点返回到侧边栏
- 侧边栏当前选中的菜单项获得焦点

### 场景 8：切换网址启用状态
**操作方式 A（在网址项上）：**
1. 聚焦在网址项上
2. 按 **Enter** 键

**操作方式 B（在按钮上）：**
1. 聚焦在按钮上
2. 按 **Enter** 键

**预期结果：**
- 网址状态切换（已添加 ↔ 未添加）
- 按钮文字切换（移除 ↔ 添加）
- 状态文字颜色和样式更新

### 场景 9：完整导航流程测试
**完整操作序列：**
1. 从侧边栏"常规"开始
2. 按 **↓** 到"网址管理"
3. 按 **右键** 进入内容区
4. 按 **↓** 移动到"抖音"
5. 按 **右键** 进入按钮
6. 按 **Enter** 切换状态
7. 按 **左键** 返回网址项
8. 按 **左键** 返回侧边栏
9. 按 **↑** 回到"常规"

**预期结果：**
- 所有步骤流畅无卡顿
- 焦点正确切换
- 样式正确更新

## 常见问题排查

### 问题 1：按右键无法进入按钮层级
**检查点：**
- 确认当前聚焦在网址项上（有视觉反馈）
- 确认 `focusedButtonIndex` 正确更新
- 检查按钮的 `tabindex` 绑定是否正确

### 问题 2：按上下键无法切换
**检查点：**
- 确认当前在正确的聚焦层级
- 检查 `availableShortcuts.value.length` 是否正确
- 验证 `focusSite` 函数逻辑

### 问题 3：焦点不按照预期移动
**检查点：**
- 检查 `focusedSiteIndex` 和 `focusedButtonIndex` 的值
- 确认 `nextTick` 中的焦点设置是否正确
- 查看控制台是否有错误信息
